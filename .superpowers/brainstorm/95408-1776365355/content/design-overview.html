Full Pipeline — Design Overview

4 phases, each building on the last. Does this structure look right?

1

Install Tools & Packages

~5 min
In DPAVO (npm):
gsap · @gsap/react · lenis · splitting · embla-carousel

mitmproxy — TLS proxy to intercept & capture assets
Playwright already in DPAVO devDeps ✓
2

Full Site Scan — incredifulls.com

~20 min automated

A Playwright script crawls every page of incredifulls.com — homepage, product pages, store finder, about. It:

  • Downloads all JS/CSS assets (gsap.min.js, ScrollTrigger.min.js, lenis.min.js, landing.css, shark.css…)
  • Extracts all GSAP timeline/ScrollTrigger calls from global.js via JS evaluation
  • Screenshots every scroll position on every page
  • Records all animation config values (duration, ease, stagger, scrub settings)
  • Saves everything to Projects/Reverse-Engineer-Website/
3

Analyze + Create Reusable Skill

permanent

From the scan output, I build a Claude Code skill that lives permanently in your ~/.claude/ setup. The skill contains:

  • Installation recipes for every animation package
  • Copy-paste code templates for all 6 animation types
  • Exact config values extracted from incredifulls (easing curves, durations, stagger values)
  • React/Next.js specific patterns (SSR safety, useGSAP hook, cleanup)
  • Invoke with /pizza-animations on any future project
4

Implement in D'Pavo — Home + Menu
